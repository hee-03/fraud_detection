핵심 문제와 데이터셋

0.17%

극단적인 클래스 불균형

전체 284,807건 중 사기 거래는 492건에 불과합니다. 단순 정확도(Accuracy) 기반 평가는 의미가 없으며, Recall과 PR-AUC 중심의 평가 전략이 필수적입니다.

V1~V28

PCA 익명화 피처

28개 피처는 PCA 변환으로 익명화되어 원본 의미가 비공개입니다. 머신러닝은 피처의 '의미'가 아닌 '패턴'을 학습하므로, 오히려 ML 입력으로 더 적합한 형태가 됩니다.

🔁 비대칭 오류 비용

FN(미탐): 사기 거래를 정상으로 판단 → 금전 피해 발생, 회수 불가
FP(오탐): 정상 거래를 사기로 판단 → 고객 불편, 전화 한 통으로 해결 가능

📐 전처리 전략

Amount와 Time을 별도 스케일러로 분리하여 scaler 혼용 오류를 방지합니다. Stratified split으로 0.17% 사기 비율을 학습/테스트에 균등 유지합니다.

왜 V1~V28을 몰라도 학습이 가능한가? — 모델은 Class=1(사기)인 행들의 숫자 조합에서 통계적 패턴을 발견합니다. "V14 ≤ -5일 때 사기가 자주 발생한다"는 상관관계는 V14의 실제 의미를 몰라도 학습됩니다. PCA 변환은 다중공선성을 제거하고 분산 기준으로 정보를 압축하므로 오히려 ML 입력으로 더 적합합니다.
